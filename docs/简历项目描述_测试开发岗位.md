# NVMe SSD 自动化测试与验证项目

## 项目背景
针对存储产品测试需求,独立设计并实现了一套完整的 NVMe SSD 自动化测试系统,覆盖功能验证、性能测试、健康监控及底层命令验证,形成可复用的测试框架和工具链。

## 技术栈
- 测试脚本开发: Python 3
- 底层工具开发: C (Linux ioctl、O_DIRECT)
- 测试工具: nvme-cli、fio、smartctl、blkdiscard
- 测试环境: Ubuntu、真实 NVMe SSD 硬件

## 核心工作内容

### 1. 测试框架设计与实现 (Python)
- 设计并实现了模块化的测试框架,包含测试编排、结果采集、日志管理和报告生成功能
- 实现了统一的命令执行封装,自动记录所有命令的执行时间、返回码、输出内容,便于问题复盘
- 开发了测试区域自适应算法,根据不同容量 SSD 自动调整测试偏移量,提升测试脚本的通用性
- 实现了结构化的测试结果管理,支持 JSON 和 Markdown 双格式输出,便于自动化分析和人工审阅

### 2. 数据一致性验证测试
- 实现了基于固定随机种子的可复现数据模式生成算法,避免使用固定 pattern (0x00/0xFF) 可能遗漏的一致性问题
- 开发了多区域裸盘写入/回读校验功能,支持配置化的偏移量、长度、块大小和随机种子
- 使用 SHA256 哈希校验确保数据完整性,在发现不一致时精确定位到字节级偏移量
- 测试覆盖盘片不同物理位置 (前端、中部、后端),验证全盘数据可靠性

### 3. NVMe 特性功能测试
- 实现了 Flush 命令测试,验证数据持久化机制是否正常工作
- 实现了 TRIM/Discard 测试,验证 discard 前后的数据写入、回读及重写流程
- 开发了 SMART 健康信息对比功能,自动提取关键字段 (critical_warning、media_errors、temperature 等)
- 实现了测试前后 SMART 数据对比分析,自动识别测试过程中新增的健康风险

### 4. 性能冒烟测试 (基于 fio)
- 集成 fio 工具,实现了顺序读写、随机读写、混合读写等多种场景的性能测试
- 开发了 fio 命令构建器,支持参数化配置 (块大小、队列深度、读写比例、测试时长等)
- 实现了 fio JSON 输出解析器,自动提取 IOPS、带宽、延迟 (平均值、P99) 等关键性能指标
- 支持单向和混合读写模式的自动识别和指标分类展示

### 5. 底层 NVMe 命令工具开发 (C)
- 基于 Linux `NVME_IOCTL_ADMIN_CMD` 接口开发了 NVMe Admin 命令测试工具
- 实现了 Identify Controller、Identify Namespace、SMART Log、Error Log 等命令的查询和解析
- 手动解析 NVMe 数据结构 (小端序读取、字段提取、ASCII 字符串处理),加深对 NVMe 协议的理解
- 工具输出格式化的设备信息,包括厂商 ID、型号、固件版本、LBA 格式、温度、磨损度等

### 6. O_DIRECT 直通 I/O 验证工具开发 (C)
- 开发了基于 O_DIRECT 的裸盘数据校验工具,绕过操作系统页缓存,直接验证设备层数据一致性
- 实现了内存对齐检查 (逻辑块大小、4K 页对齐),确保 O_DIRECT 操作的合法性
- 使用 xorshift64* 算法生成可复现的测试数据模式,使用 FNV-1a 哈希进行快速校验
- 实现了完整的写入-同步-回读-对比流程,在发现不一致时输出详细的偏移量和哈希值

### 7. 测试报告与日志系统
- 实现了自动化的 Markdown 测试报告生成,包含测试概览、各项测试结果、SMART 对比、告警信息
- 开发了结构化的 JSON 日志系统,记录所有命令执行细节和测试结果,支持后续数据分析
- 实现了测试产物管理,自动保存原始 JSON 输出、命令日志、测试报告到时间戳目录
- 支持测试状态分级 (PASS/WARN/FAIL/SKIP),便于快速识别测试问题

## 项目成果
- 在真实 NVMe SSD 硬件上完成了完整的测试验证,未发现数据一致性错误或介质错误
- 形成了可复用的测试脚本 (1000+ 行 Python) 和底层工具 (600+ 行 C)
- 建立了完整的测试文档体系,包括执行指南、测试项详解、NVMe 基础知识等
- 测试框架支持参数化配置,可快速适配不同容量和型号的 NVMe SSD
