# 简历项目描述

## 项目名称

NVMe SSD 自动化测试与低层验证项目

## 项目背景

为系统学习半导体存储测试相关工作流程，自行购买 `Lexar SSD ARES PRO 1TB NVMe M.2 SSD` 作为测试对象，基于 Ubuntu 环境搭建 NVMe SSD 基础验证项目，并在 Ubuntu 与树莓派环境中执行测试，围绕 SSD 的功能正确性、基础性能、TRIM/Flush、健康状态监控以及低层命令通路开展验证。

## 项目职责

1. 独立完成测试项目选题、方案设计、测试项拆分与工具选型。
2. 使用 Python 开发 SSD 自动化测试主控脚本，负责测试编排、结果采集、日志归档和 Markdown 报告生成。
3. 使用 C 开发 NVMe 低层测试工具，完成 Admin 命令验证及 `O_DIRECT` 直通 I/O 校验。
4. 基于真实硬件完成 Ubuntu 与树莓派双环境测试执行，并结合 SMART、性能结果和底层日志进行分析。

## 技术方案

1. 使用 `Python` 构建测试主流程，调用 `nvme-cli`、`smartctl`、`fio`、`blkdiscard` 等工具完成自动化执行。
2. 使用 `C` 结合 Linux `NVME_IOCTL_ADMIN_CMD` 实现 `Identify Controller`、`Identify Namespace`、`SMART Log`、`Error Log`、`Flush` 等 NVMe Admin 命令验证。
3. 使用 `C` 开发 `O_DIRECT` 裸盘校验工具，完成对齐内存申请、direct I/O 写入/回读及数据一致性比对。
4. 使用 JSON 配置文件管理测试参数，支持按 offset、块大小、测试长度、混合读写比例等维度调整测试计划。

## 主要测试内容

1. SSD 基础信息识别与健康信息采集，包括型号、固件版本、namespace、SMART 状态等。
2. 裸盘 pattern 写入/回读校验，在多个 LBA 区域验证数据一致性。
3. `Flush` 测试，验证 NVMe Flush 命令执行路径及数据持久化相关行为。
4. `TRIM/Discard` 测试，验证 discard 后重写与回读流程是否正常。
5. 基于 `fio` 的顺序读写、随机读写、混合读写性能冒烟测试。
6. 基于 C 工具的 NVMe Admin 命令验证及 `O_DIRECT` 直通 I/O 校验。
7. 测试前后 SMART 对比分析，重点关注 `critical_warning`、`media_errors`、`num_err_log_entries` 等字段变化。

## 项目成果

1. 独立搭建了一套可运行的 NVMe SSD 自动化测试项目，形成 Python 脚本、C 工具、配置文件和测试文档的完整闭环。
2. 在真实硬件 `Lexar SSD ARES PRO 1TB` 上完成功能测试、性能测试及低层命令验证，具备实际动手经验。
3. 在 Ubuntu 与树莓派环境中执行测试，积累了不同 Linux 平台下的 SSD 测试实践经验。
4. 实现测试报告自动输出，能够沉淀原始日志、结构化结果和 Markdown 分析报告，便于复盘和汇报。
5. 对 NVMe SSD 的基础测试方法、Linux 块设备 I/O、`fio` 参数、SMART 指标和 `O_DIRECT` 直通 I/O 机制建立了系统理解。

# kiro

NVMe SSD 自动化测试与验证工具

项目背景： 针对存储产品测试需求，开发了一套完整的NVMe SSD自动化测试框架，覆盖功能验证、性能评估、健康监控等核心测试场景。

核心功能与技术实现：

• 基于 Python 开发自动化测试框架，实现测试编排、结果采集与报告生成。采用 os 模块的 pwrite/pread 实现裸盘直接 I/O 操作，通过固定随机种子生成可复现的测试数据模式，完成多区域写入/回读一致性校验，使用 SHA256 哈希算法验证数据完整性。

• 集成 nvme-cli、smartctl、fio、blkdiscard 等 Linux 存储工具，通过 subprocess 模块调用外部命令并解析 JSON 格式输出，实现设备信息采集、SMART 健康状态监控、Flush 持久化测试、TRIM/Discard 功能验证以及顺序/随机/混合读写性能冒烟测试。

• 使用 C 语言开发底层 NVMe 管理工具，通过 ioctl 系统调用和 NVME_IOCTL_ADMIN_CMD 接口直接下发 NVMe Admin 命令（Identify Controller、Identify Namespace、SMART Log、Error Log），实现对控制器和命名空间的低层信息查询与健康状态检测。

• 基于 C 语言实现 O_DIRECT 数据校验工具，使用 posix_memalign 进行内存对齐，通过 O_DIRECT | O_SYNC 标志绕过页缓存直接访问裸盘，采用 xorshift64* 伪随机算法生成测试模式，结合 FNV-1a 哈希算法完成写入/回读一致性验证，确保直通 I/O 路径的数据可靠性。

• 实现智能测试区域管理机制，根据设备容量动态调整测试偏移量，避免因不同容量 SSD 导致测试失败。设计 SMART 前后对比检查，监控 critical_warning、media_errors、num_err_log_entries 等关键指标变化，及时发现测试过程中的健康风险。

• 开发完整的测试报告系统，自动生成 Markdown 格式测试报告和 JSON 结构化数据，包含测试结论、详细指标、告警信息、命令执行日志等，支持测试结果的可视化展示与问题复盘。

难点问题： 在实现裸盘数据校验时，需要确保测试数据模式的可复现性。通过使用固定随机种子的伪随机数生成器，保证写入和验证阶段生成完全相同的数据序列，成功解决了数据一致性验证的可靠性问题。

技术栈： Python、C、Linux 系统编程、NVMe 协议、存储测试工具（nvme-cli、fio、smartctl）、JSON 数据处理、Makefile













NVMe SSD 自动化测试与验证平台

项目背景

针对存储产品测试需求，独立设计并实现了一套完整的 NVMe SSD 自动化测试框架，覆盖功能验证、性能评估、健康监控等核心测试场景，可生成结构化测试报告用于产品质量评估与问题复盘。

技术栈

Python 3、C 语言、Linux 系统编程、NVMe 协议、fio 性能测试工具、nvme-cli、smartctl、JSON 数据处理

核心功能与技术实现

测试框架设计与实现：基于 Python 开发自动化测试主框架，实现测试编排、结果采集、日志落盘和 Markdown 报告自动生成；通过 JSON 配置文件实现测试计划的灵活配置，支持多区域、多参数的测试场景组合

数据一致性验证：使用 Python 实现基于固定随机种子的可复现 pattern 数据生成算法，通过 SHA256 哈希校验完成裸盘多区域写入/回读验证；使用 C 语言结合 O_DIRECT 标志实现绕过页缓存的直通 I/O 校验，采用 FNV-1a 哈希算法和 posix_memalign 确保内存对齐，验证底层数据路径的可靠性

NVMe 协议层测试：使用 C 语言通过 ioctl 系统调用直接下发 NVMe Admin 命令（Identify Controller、Identify Namespace、SMART Log、Error Log、Flush），验证管理命令通路和设备响应正确性；实现 TRIM/Discard 功能测试，验证空间回收机制的有效性

性能评估与分析：集成 fio 工具完成顺序读写（128KB）、随机读写（4KB）、混合读写（70/30）等多种负载模式的性能冒烟测试；解析 fio JSON 输出提取 IOPS、带宽、平均延迟、P99 尾延迟等关键指标，识别性能瓶颈

健康状态监控：通过 smartctl 和 nvme-cli 采集测试前后的 SMART 数据快照，自动对比 critical_warning、media_errors、temperature、percentage_used 等关键健康指标，及时发现测试过程中的异常变化

测试报告与可追溯性：自动生成包含测试结论、详细数据、告警信息的 Markdown 报告和结构化 JSON 摘要；记录所有命令执行日志（命令、返回码、耗时、输出），确保测试过程完全可复盘

项目成果

成功验证 1TB NVMe SSD 的基础功能、数据一致性、TRIM/Flush 机制和性能表现，测试过程未发现介质错误或健康异常
测试框架支持不同容量 SSD 的自适应测试区域调整，提升了测试脚本的通用性和鲁棒性
通过 Python 与 C 的混合编程，既保证了测试框架的开发效率，又实现了对底层协议和 I/O 路径的深度验证
技术亮点

掌握 Linux 块设备编程（O_DIRECT、O_SYNC、fsync、pread/pwrite）和内存对齐要求
熟悉 NVMe 协议栈，能够通过 ioctl 直接操作 NVMe Admin 命令
具备完整的测试工程能力：从测试设计、脚本开发、执行验证到报告输出的全流程实践经验
理解存储测试中的关键指标（IOPS、带宽、延迟分布、SMART 健康参数）及其业务意义
